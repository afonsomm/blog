Deployment - Clean Data > Recomendação de treino

This script consists of 3 main steps:

  1. Load from the database, i.e., students’ test results, tests’ reference values, and previously cleaned data (resulting from this same script).
  2. Clean, i.e., filtering out loaded records with invalid information, transforming existing features, and creating new ones. The steps that contain “audit” in the diagram also create a DataFrame to log the changes made to the original data.
  3. Aggregate to keep each test’s “Best” or “Latest” value, which is chosen on the next step depending on the use case (on this one, it’s the “Latest”).

Load

  • The Raw Data is loaded from the FITescola database, containing all the students’ results for the given school year, as originally inserted.
  • The Reference Values are loaded from a table in the FITescola database and contain the bounds defining each test’s health zones for a given gender and age.
  • The Cleaned Results are loaded from a table in the FOCUS database and contain past cleaned data for the given school year.

Clean

In this part, the students’ data is filtered out if they have invalid data, their test values are corrected, and new features are obtained from the corrected tests.

  1. The Mile and Shuttle values are substituted for their “raw value”, corresponding to the time and number of laps, respectively.
  2. Students’ data is filtered out if it has an invalid school year. Subsequently, if their age is outside the defined bounds for their school year, their data is also removed.
  3. The Spark DataFrame is pivoted, going from 1 row per (“student id”, “test date”, “test name”), to 1 row per (“student id”, “test date”), meaning each test has its column.
  4. Students’ Fat Mass is calculated using predefined formulas by gender, using their Calf Skinfold and Tricep Skinfold measurements. The calculated values are then used to fill in missing values from the original Fat Mass column.
  5. A sequence of cleaning operations is applied to the different tests (defined in init.py -> clean_tests() -> clean_func_params_seq). The various operations are defined as follows:
    1. Clean by bounds - a single test’s values are either removed or transformed between or outside a certain range. The values may be cleaned for a given age range and gender.
    2. Clean line metric X equals metric Y—two tests’ values are optionally removed if they are equal and either above or below a certain value. The data can also be optionally clean for a given age range and gender.
    3. Switch tests - two tests’ values are either removed or switched for each student if they are possibly switched in the database and depending on the number of samples they have.
    4. Clean area line metric X equals metric Y - two tests’ values are optionally removed above or below specific values if their values are either above or below a line between 2 points defined by both tests’ values and optionally for a given age range and gender.
    5. Remove consecutive equal measurements - a single test’s values are removed, if a student has more than a given number of consecutive equal measurements.
  6. Calculate BMI based on the corrected weight and height tests’ values.

Besides the steps above, after the results aggregation, explained below, the VO2 Max is calculated from the final mile and shuttle values by obtaining the VO2 for the Mile and Shuttle tests, joining the reference values for this parameter, and keeping the VO2 from both tests with the best value, meaning the one with the highest distance to the worst healthy bound (in this case the minimum bound).

Aggregate

In this part, the corrected results from the Raw Data are aggregated along with the loaded Cleaned Results.

Before each step of this phase, the processed DataFrame is melted, so we go back to having one line per (“student id”, “test date”, “test name”), and joined to the Reference Values.

  1. The distances to the worst healthy bound and the healthy zone are obtained. These are used to aggregate the students’ results and, in the following pipeline steps, get the score (health zone) for each test, respectively.
  2. The distance to the worst healthy bound is obtained by calculating the difference between each test’s values and the closest worst healthy bound so that a higher distance means better results. For instance, there are three zones for the Abs test, i.e., PM -> ZS -> PA, so this distance is the test value subtracted from the minimum value of the healthy zone (ZS). However, for the BMI, there are two zones, i.e., PM -> ZS -> PM, so this distance is calculated to the closest healthy bound in a way that it is negative outside the healthy zone and positive inside.
  3. The distance to the healthy zone is calculated by obtaining the healthy zone ranges for each test (max bound - min bound) and transforming the distance to the worst healthy bound, making it positive if it is higher than the range, 0 if inside the healthy zone, and keeping the negative values.
  4. The Cleaned Results are concatenated to the corrected Raw Data results. The VO2 Max is filtered out of the first one since it is calculated after the aggregation.
  5. The results are aggregated to obtain each student’s “Best” results and, if it is the current year, also the “Latest.” 
  6. The first (“Best”) corresponds to the results with the highest distance to the worst healthy bound unless it is a Body Composition test, in which case the latest value is kept.
  7. The second (“Latest”) corresponds to each student’s current state, keeping the last test results.
